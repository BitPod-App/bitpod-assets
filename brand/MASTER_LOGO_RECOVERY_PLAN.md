# BitPod Master Logo Recovery Plan

Status: Active
Date: 2026-04-10

## Non-negotiable premise

The next true BitPod SVG master should be drawn manually by a skilled human designer.

Not by:

- automatic vector tracing
- current wrapper SVG reuse
- export cleanup alone
- wishful thinking dressed as asset management

## Trusted recovery input

Use this as the primary visual authority:

- `brand/logo/png/bitpod-logo-avatar-square-color.png`

Use the current SVG wrappers only as secondary reference material for sizing, placement, and rough packaging context.

## Phase 1 — Quarantine false canon

Goal: stop the repo from implying that the current SVG set is a real master system.

Deliverables:

- recovery notice naming the trusted source of truth
- preserved catastrophe docs
- explicit statement that current SVGs are non-canonical wrappers

## Phase 2 — Create the true primary SVG master

Goal: produce one approved, editable, geometry-native master.

Requirements:

- manual redraw in Figma or Illustrator
- optical fidelity to the trusted raster reference
- clean curves and anchor discipline
- no embedded raster payload pretending to be vector
- one approved primary master before any export explosion begins

Recommended owner:

- a strong brand or identity designer with precision vector skill

## Phase 3 — Derive the official export system

Goal: derive exports from truth instead of deriving truth from exports.

Deliverables:

- approved primary SVG master
- approved PNG exports
- lockup variants only if explicitly needed
- mini guide stating which files are official and which are not
- retirement note for the recovery-era wrapper system

## Success condition

BitPod should end this process with:

1. one true master
2. one small official export family
3. one clear usage guide
4. zero ambiguity about where logo truth lives
